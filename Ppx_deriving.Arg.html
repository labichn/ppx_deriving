Module Ppx_deriving.Arg

module Arg: sig .. end
Ppx_deriving.Arg contains convenience functions that extract constants from AST fragments, to be used when parsing options or [@attributes] attached to types, fields or constructors.

The ~name argument is used in error messages and should receive the name of the deriving plugin, e.g. "Show".

val int : Parsetree.expression -> [ `Error of string | `Ok of int ]
int expr extracts an integer constant from expr, or returns `Error "integer" if expr does not contain an integer constant.
val string : Parsetree.expression -> [ `Error of string | `Ok of string ]
string expr extracts a string constant from expr, or returns `Error "string" if expr does not contain a string constant.
val enum : string list -> Parsetree.expression -> [ `Error of string | `Ok of string ]
enum values expr extracts a polymorphic variant constant from expr, or returns `Error "one of: `a, `b, ..." if expr does not contain a variant included in values.
val payload : name:string ->
       (Parsetree.expression -> [ `Error of string | `Ok of 'a ]) ->
       Parsetree.attribute option -> 'a option
payload conv attr extracts the expression from attr and converts it with conv, raising Location.Error if attr is not a structure with a single expression or conv fails; or returns None if attr is None.

Example usage: 

let kind =

  match Ppx_deriving.attr ~prefix:"index" "kind" pcd_attributes |>

        Ppx_deriving.Arg.(payload ~name:"Ix" (enum ["flat"; "nested"])) with

  | Some idx -> idx | None -> acc

in ..
